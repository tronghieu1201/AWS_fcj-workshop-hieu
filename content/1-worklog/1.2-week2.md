## 3. Content

- AWS Networking:
    - Learned the basic networking components in AWS, including Amazon VPC, subnet, route table, and Internet Gateway.
    - Practiced creating a VPC and enabling DNS resolution and DNS hostnames for the VPC.
    - Configured subnets, route tables, and Internet Gateway to build a basic network model in AWS.
    - Used VPC Resource Map to observe the overall network resources and the relationships between them.

- IAM Role:
    - Learned how to grant EC2 access to AWS services using an IAM Role.
    - Compared the use of direct Access Keys with the use of IAM Roles.
    - Practiced creating an IAM Role, attaching an S3 policy, and assigning the role to an EC2 instance.
    - Verified that the EC2 instance used credentials from the IAM Role instead of manually configured Access Keys.

- Route 53:
    - Learned the Hybrid DNS model between the AWS environment and an internal/on-premises DNS system.
    - Created the lab infrastructure using CloudFormation, including VPC, subnet, Security Group, NAT Gateway, Load Balancer, Auto Scaling Group, and EC2 RDGW.
    - Successfully connected to the EC2 RDGW instance using Remote Desktop to verify the lab environment.
    - Deployed AWS Managed Microsoft AD with the domain `onprem.example.com` to simulate an on-premises DNS system.
    - Created a Route 53 Resolver Outbound Endpoint to forward DNS queries from the VPC to the Microsoft AD DNS server.
    - Created a Forward Resolver Rule for the domain `onprem.example.com`.
    - Created a Route 53 Resolver Inbound Endpoint to support DNS queries from outside into the AWS environment.
    - Verified DNS resolution using the `nslookup` command and tested connectivity using the `ping` command.

- AWS CLI:
    - Learned how to use AWS CLI to interact with AWS services through the command line.
    - Checked AWS CLI on EC2, configured the region, and verified that credentials were obtained from the IAM Role.
    - Practiced using AWS CLI with services such as S3, SNS, IAM, VPC, and EC2.
    - Created, checked, and cleaned up AWS resources using the command line after completing the lab.
