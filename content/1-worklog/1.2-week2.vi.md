## 3. Nội dung 
- Mạng AWS:
    - Tìm hiểu các thành phần mạng cơ bản trong AWS gồm Amazon VPC, subnet, route table và Internet Gateway.
    - Thực hành tạo VPC, bật DNS resolution và DNS hostnames cho VPC.
    - Cấu hình subnet, route table và Internet Gateway để xây dựng mô hình mạng cơ bản trong AWS.
    - Sử dụng VPC Resource Map để quan sát tổng quan các tài nguyên mạng và mối liên kết giữa chúng.
- IAM Role:
    - Tìm hiểu cách cấp quyền cho EC2 truy cập dịch vụ AWS bằng IAM Role.
    - So sánh cách sử dụng Access Key trực tiếp với cách sử dụng IAM Role.
    - Thực hành tạo IAM Role, gán policy S3 và attach role vào EC2 instance.
    - Kiểm tra EC2 sử dụng credentials từ IAM Role thay vì nhập Access Key trực tiếp.
- Route 53:
    - Tìm hiểu mô hình Hybrid DNS giữa môi trường AWS và hệ thống DNS nội bộ/on-premise.
    - Khởi tạo hạ tầng lab bằng CloudFormation, bao gồm VPC, subnet, Security Group, NAT Gateway, Load Balancer, Auto Scaling Group và EC2 RDGW.
    - Kết nối thành công đến EC2 RDGW bằng Remote Desktop để kiểm tra môi trường thực hành.
    - Triển khai AWS Managed Microsoft AD với domain `onprem.example.com` để mô phỏng hệ thống DNS on-premise.
    - Tạo Route 53 Resolver Outbound Endpoint để chuyển tiếp truy vấn DNS từ VPC đến DNS server của Microsoft AD.
    - Tạo Resolver Rule dạng Forward cho domain `onprem.example.com`.
    - Tạo Route 53 Resolver Inbound Endpoint để hỗ trợ truy vấn DNS từ bên ngoài vào môi trường AWS.
    - Kiểm tra kết quả phân giải DNS bằng lệnh `nslookup` và kiểm tra kết nối bằng lệnh `ping`.
- AWS CLI:
    - Tìm hiểu cách sử dụng AWS CLI để thao tác với các dịch vụ AWS bằng dòng lệnh.
    - Kiểm tra AWS CLI trên EC2, cấu hình region và xác minh credentials được lấy từ IAM Role.
    - Thực hành sử dụng AWS CLI với các dịch vụ S3, SNS, IAM, VPC và EC2.
    - Tạo, kiểm tra và dọn dẹp tài nguyên AWS bằng command line sau khi hoàn thành lab.
