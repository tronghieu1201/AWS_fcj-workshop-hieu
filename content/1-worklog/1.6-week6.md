## 3. Content

- AWS Lambda and Slack:
  - Learned how to use AWS Lambda to automate EC2 operational tasks.
  - Practiced creating an IAM Role that allows Lambda to work with EC2 and CloudWatch Logs.
  - Added tags to EC2 so Lambda can identify the correct instance to start or stop.
  - Configured a Lambda function to automatically start/stop EC2 based on usage needs.
  - Integrated Slack Webhook to send notifications when Lambda actions are completed successfully.

- CloudWatch and Grafana:
  - Learned how to monitor AWS resources through CloudWatch metrics.
  - Created an EC2 instance and configured an IAM Role so Grafana could read monitoring data.
  - Installed Grafana on EC2 and connected Grafana with CloudWatch as a data source.
  - Created a basic dashboard to monitor CPU, network, and resource status.
  - Became familiar with visualizing monitoring data instead of only viewing metrics directly in the AWS Console.

- AWS Systems Manager Session Manager:
  - Learned how to securely access EC2 using Session Manager without an SSH key.
  - Practiced configuring an IAM Role for EC2 to use AWS Systems Manager.
  - Connected to public/private EC2 instances through Session Manager on the AWS Console.
  - Understood the security benefit of Session Manager by reducing the need to open SSH port 22.
  - Checked session logs to review access history and actions performed on the instance.

- AWS CloudFormation:
  - Learned the concept of Infrastructure as Code in AWS infrastructure deployment.
  - Practiced writing and using CloudFormation templates to create resources automatically.
  - Created a CloudFormation Stack, checked deployment status, and reviewed the created resources.
  - Updated or deleted stacks to manage the resource lifecycle.
  - Understood how CloudFormation helps deploy infrastructure consistently, repeatedly, and more easily than manual configuration.
