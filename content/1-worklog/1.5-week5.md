## 3. Content

- Amazon S3 Security:
  - Created the lab environment using CloudFormation for the S3 Security hands-on lab.
  - Modified the template to use EC2 `t3.micro`, which is suitable for the practice account.
  - Connected to the lab EC2 instance using EC2 Instance Connect and worked with AWS CLI.
  - Created access keys for `SID-lab-user1` and `SID-lab-user2`, then configured CLI profiles `user1` and `user2`.
  - Checked access permissions to the S3 Security Lab bucket and sample data under `app1/` and `app2/`.
  - Configured a bucket policy to require S3 access through HTTPS.
  - Cleaned up resources after completing the lab.

- IAM Cost Control:
  - Learned how to use IAM Policy to limit AWS resource usage.
  - Practiced restricting resource creation by Region, EC2 instance type, and EBS volume type.
  - Became familiar with IAM Condition to control costs and prevent resources from being created outside the allowed scope.

- Tags and Resource Groups:
  - Learned how to use Tags to classify AWS resources by environment, owner, or usage purpose.
  - Practiced adding tags, filtering resources by tag, and managing resources by group.
  - Learned how Resource Groups can group resources with the same tags, making management and monitoring easier.
